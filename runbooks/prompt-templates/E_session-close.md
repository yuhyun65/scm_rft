# 템플릿 E — 세션 종료 프롬프트

> **사용 시점:** 매 작업 세션의 마지막 프롬프트
> **목적:** 종료 조치를 표준화하고, 다음 세션 시작 직전 상태를 자동 요약
> **절감 효과:** 다음 세션 시작 시 환경 확인 Q&A 0회 (템플릿 A의 "전 세션 종료 상태" 자동 생성)

---

## 프롬프트 본문 (복사해서 사용)

```
오늘 작업을 종료합니다. 다음을 순서대로 처리해줘.

【종료 조치 (순서대로 자동 실행)】
1. 미커밋 변경사항 확인
   - doc/QnA_보고서.md 및 오늘 작업 관련 파일 기준 브랜치에 커밋/푸시
   - untracked 파일 중 커밋 대상: {있으면 명시 / 없으면 "없음"}
   - 커밋 제외 확인: services/*/bin/, runbooks/evidence/*.log, migration/reports/*.out.txt

2. 실행 중인 서비스 종료
   - 포트 8081 / 8082 / 18080 점유 프로세스 종료
   - Gradle daemon 정지: .\gradlew.bat --stop

3. Docker 인프라 정리
   - scripts/staging-down.ps1 실행 (컨테이너/네트워크 down)
   - docker ps 결과 실행 중 컨테이너 없음 확인

4. 워크트리 정리
   - 머지 완료된 작업 워크트리 있으면 삭제: git worktree remove --force {경로}
   - 원격 브랜치도 삭제: git push origin --delete feature/scm-{완료된번호}-...

5. 다음 세션 시작점 요약 출력
   (아래 형식으로 3~5줄 요약 — 다음 세션의 템플릿 A "전 세션 종료 상태"에 복사해 사용)
```

---

## 출력 형식 (자동 생성 요약)

Codex가 종료 조치 완료 후 아래 형식으로 요약을 출력하면, 그 내용을 다음 세션 템플릿 A에 그대로 붙여넣으면 됩니다.

```
【다음 세션 시작점】
- 마지막 머지 PR : #{N}  (SCM-{XXX} {한 줄 설명})
- 현재 브랜치    : {feature/scm-XXX-... 또는 feature/to-be-dev-env-bootstrap}
- 최신 커밋      : {7자리 해시} - {커밋 메시지}
- 다음 착수 대상 : SCM-{XXX} — {한 줄 설명}
- 미처리 항목    : {없음 / 있으면 내용}
- 환경 상태      : Docker DOWN / 서비스 포트 전부 FREE / 워킹트리 {clean or 상태}
```

---

## 세션 중간 저장 프롬프트 (장시간 작업 중 중간 커밋이 필요할 때)

```
현재 진행 중인 작업을 중간 저장해줘.

1. 현재 변경된 파일 목록 확인 (git status)
2. 의미 있는 단위로 커밋 (WIP 표시 포함)
   커밋 메시지 형식: "wip(scm-{XXX}): {진행 내용 한 줄}"
3. 원격 브랜치에 푸시
4. 현재까지 완료된 항목 / 남은 항목을 3줄로 요약해줘.
```

---

## 이슈/PR 정리 전용 프롬프트 (머지 후 정리만 필요할 때)

```
다음 정리 작업을 실행해줘.

1. PR #{N} 머지 상태 확인 (gh pr view {N})
2. 머지 완료 확인 시:
   - Issue #{M} close: gh issue close {M} --comment "Merged via PR #{N}"
   - 기준 브랜치 동기화: git checkout feature/to-be-dev-env-bootstrap && git pull --ff-only
   - 작업 브랜치 삭제 (로컬 + 원격): feature/scm-{XXX}-...
   - 워크트리 있으면 제거: git worktree remove --force {경로}
3. doc/QnA_보고서.md 업데이트 후 커밋/푸시
4. 현재 open 이슈/PR 목록 요약 출력
```

---

## 적용 전/후 비교

| 구분 | 기존 방식 (Q&A 횟수) | 최적화 후 |
|---|---|---|
| 종료 조치 지시 | Q_n (1회) | 1회 (자동 처리 포함) |
| 종료 처리 진행 승인 | Q_n+1 (1회) | 자동 처리 선언으로 제거 |
| 다음 진행사항 요청 | Q_n+2 (1회) | 종료 프롬프트에 통합 |
| **합계** | **3회** | **1회** |

다음 세션 시작 Q&A도 함께 절감:

| 구분 | 기존 방식 | 최적화 후 |
|---|---|---|
| 다음 세션 환경 확인 | 2~3회 | 0회 (종료 요약 → 템플릿 A에 복붙) |
