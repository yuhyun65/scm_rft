# 템플릿 C — 병렬 이슈 실행 프롬프트

> **사용 시점:** 상호 독립적인 이슈 2~3개를 동시에 처리할 때
> **목적:** 직렬 대기 제거 + 의존 관계 명시로 순서 오류 방지
> **절감 효과:** 병렬 가능 구간에서 전체 작업 시간 30~50% 단축

---

## 의존 관계 판단 기준

병렬 실행이 가능한 경우:

| 조건 | 예시 |
|---|---|
| 서로 다른 서비스 디렉터리를 수정 | auth vs member |
| 공유 파일(shared/contracts, build.gradle)을 수정하지 않음 | 독립 도메인 API 구현 |
| DB 스키마가 겹치지 않음 | 다른 Flyway 버전 번호 사용 |

병렬 실행이 불가한 경우(직렬 필수):

| 조건 | 예시 |
|---|---|
| 한 이슈의 산출물을 다른 이슈가 의존 | Gateway가 Auth 토큰 검증 API를 호출해야 함 |
| 동일 파일을 양쪽이 수정 | 같은 build.gradle 또는 settings.gradle |
| 기준 브랜치 상태가 정합해야 착수 가능 | Flyway V5 적용 후 V6 착수 |
